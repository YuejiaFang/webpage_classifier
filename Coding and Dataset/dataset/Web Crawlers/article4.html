Web Crawler

The Web Crawler application is an automated program that methodically tracks Internet pages and collects data. It also compares the size and contents of a file against the version of that file stored in InfoSphereÂ® BigInsightsâ¢.

You can use the Web Crawler application to:
  • Gain business intelligence into how competitors describe their products or plans
  • Locate relevant information about disease diagnoses and drug trials

The InfoSphere BigInsights Web Crawler application is based on Apache Nutch. The application supports scheduled crawling and the use of crawled data by multiple applications. After you run the Web Crawler, you can import the output directory of the Web Crawler application into BigSheets to read results or link that data to an existing sheet.

Specify values for the following parameters as indicated:

URLs
A list of URLs for the Web Crawler application to crawl
The URLs should be fully qualified names (FQNs).
Filters
A new line delimited list of URL filters. Entries begin with a + to denote if the entry is a match for a URL, or - to denote that the URL does not contain the filter value. If neither + or - are found at the beginning of the entry, the value defaults to +.
Containing filter
To retrieve all results from crawling a specific URL, use a + filter without including any URLs. For example, specify the following filter if you want to crawl a URL and retrieve all results:
+
Restriction: You cannot use a containing filter to crawl only the URLs that you specify. For example, using the following filter returns no results:
+Adamson
When using a containing filter, you can include the root URL as part of the filter. For example, you might want to crawl the following URL for patents and include entries that contain "Adamson".
http://www.ibm.com/software/ebusiness/jstart/bigsheets/demo/Patents.html
Use the following filter to crawl all URLs under the root URL, including entries that contain "Adamson". This filter returns the same results as if you specified a + without including a root URL.
+http://www.ibm.com
+Adamson
Exclusion filter
If you use an exclusion filter, you must include the criteria that you want to exclude, preceded by -, and then include +. at the end of your filter. For example, to crawl the previous URL and exclude entries that contain "Adamson" and "Haas", use the following filter:
-Adamson
-Haas
+.
Output directory
The path of the output directory in the /<dir_name> format
Maximum crawl depth
The link depth from the root page for the Web Crawler application to crawl
Maximum pages per crawl depth
The maximum number of pages that will be retrieved at each depth level

Examples

For example, the following parameter values return data only from URLs that contain the ibm string, like http://www.ibm.com/:
  • URL: http://www.ibm.com/
  • Filters: +ibm
  • Output directory: /tmp/crawler
  • Crawl depth: 2
  • Pages: 10
Alternatively, the following parameter values return data only from URLs that contain the developerworks string, like http://www.ibm.com/developerworks:
  • URL: http://www.ibm.com/developerworks/
  • Filters: +developerworks
  • Output directory: /tmp/crawler
  • Crawl depth: 2
  • Pages: 10
